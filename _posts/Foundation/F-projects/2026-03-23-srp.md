# Refactoring the Citation Generator with SRP & API Chaining
 
After studying the Single Responsibility Principle (SRP), it's clear that several of the generator's core functions were doing too many things at once: making them hard to test, hard to debug, and hard to reuse.
 
---
 
## What We're Applying
 
1. **Single Responsibility Principle (SRP)** — each function does ONE thing and has ONE reason to change
2. **API Chaining** — sequential async operations where each step depends on the previous, with a single `.catch()` at the end
3. **Orchestrator vs. Worker** — separating the "coordinator" function from the functions that do the actual work
 
---
 
## Problem 1: `generate()`
 
### Before — Five Responsibilities in One Function
 
```javascript
function generate() {
    // Responsibility 1: Collect input from form fields
    const payload = {
        author: safe(authorEl.value),
        date: safe(dateEl.value),
        title: safe(titleEl.value),
        source: safe(sourceEl.value),
        url: safe(urlEl.value)
    };
 
    // Responsibility 2: Validate fields and highlight missing ones
    const missing = [];
    const fields = [
        { el: authorEl, key: 'author', label: 'Author' },
        { el: titleEl, key: 'title', label: 'Title' },
        { el: sourceEl, key: 'source', label: 'Source / Website' },
        { el: dateEl, key: 'date', label: 'Publication Date' }
    ];
    fields.forEach(f => f.el.classList.remove('missing'));
    fields.forEach(f => {
        if (!payload[f.key]) {
            missing.push(f.label);
            f.el.classList.add('missing');
        }
    });
 
    // Responsibility 3: Choose formatting function and build citation
    const style = styleEl.value;
    let citation = '';
    if (style === 'mla') citation = fmtMLA9(payload);
    else if (style === 'chicago') citation = fmtChicago(payload);
    else citation = fmtAPA(payload);
 
    // Responsibility 4: Display citation to DOM
    outTextEl.innerHTML = citation;
 
    // Responsibility 5: Build and display parenthetical citation
    const parentheticalEl = document.getElementById('cite-parenthetical');
    const parenthetical = buildParenthetical({ ...payload, style });
    if (parentheticalEl) {
        parentheticalEl.innerHTML = parenthetical
            ? `<span class="citation-output-label">Parenthetical Citation:</span>${parenthetical}`
            : '';
    }
 
    // Responsibility 6: Show or hide warning messages
    const warningEl = document.getElementById('cite-warning');
    if (missing.length > 0) {
        warningEl.innerHTML = `⚠️ Missing citation info: <b>${missing.join(', ')}</b>`;
        warningEl.style.display = 'block';
    } else {
        warningEl.style.display = 'none';
    }
 
    return citation;
}
```
 
**Problems:**
- If you want to reuse validation somewhere else, you can't — it's stuck inside `generate()`
- Testing citation formatting requires setting up DOM elements
- Changing the warning UI means editing the same function that handles formatting logic
- Six separate responsibilities crammed into one place

### After — Each Function Does ONE Thing
 
```javascript
// Responsibility 1: Collect form data into a plain object
function collectFormPayload() {
    return {
        author: safe(authorEl.value),
        date: safe(dateEl.value),
        title: safe(titleEl.value),
        source: safe(sourceEl.value),
        url: safe(urlEl.value)
    };
}
 
// Responsibility 2: Validate payload and return missing field labels
function validatePayload(payload) {
    const fields = [
        { el: authorEl, key: 'author', label: 'Author' },
        { el: titleEl, key: 'title', label: 'Title' },
        { el: sourceEl, key: 'source', label: 'Source / Website' },
        { el: dateEl, key: 'date', label: 'Publication Date' }
    ];
 
    fields.forEach(f => f.el.classList.remove('missing'));
 
    const missing = [];
    fields.forEach(f => {
        if (!payload[f.key]) {
            missing.push(f.label);
            f.el.classList.add('missing');
        }
    });
 
    return missing; // returns array of missing field names
}
 
// Responsibility 3: Format a citation based on style — pure logic, no DOM
function formatCitation(payload, style) {
    if (style === 'mla') return fmtMLA9(payload);
    if (style === 'chicago') return fmtChicago(payload);
    return fmtAPA(payload);
}
 
// Responsibility 4: Display the citation and parenthetical to the DOM
function displayCitation(citation, parenthetical) {
    outTextEl.innerHTML = citation;
 
    const parentheticalEl = document.getElementById('cite-parenthetical');
    if (parentheticalEl) {
        parentheticalEl.innerHTML = parenthetical
            ? `<span class="citation-output-label">Parenthetical Citation:</span>${parenthetical}`
            : '';
    }
}
 
// Responsibility 5: Show or hide validation warnings
function displayValidationWarning(missing) {
    const warningEl = document.getElementById('cite-warning');
    if (missing.length > 0) {
        warningEl.innerHTML = `⚠️ Missing citation info: <b>${missing.join(', ')}</b>`;
        warningEl.style.display = 'block';
    } else {
        warningEl.style.display = 'none';
    }
}
 
// ORCHESTRATOR: Coordinates the generate workflow
function generate() {
    const payload = collectFormPayload();
    const missing = validatePayload(payload);
    const style = styleEl.value;
    const citation = formatCitation(payload, style);
    const parenthetical = buildParenthetical({ ...payload, style });
 
    displayCitation(citation, parenthetical);
    displayValidationWarning(missing);
 
    return citation;
}
```
 
**What changed:**
- `formatCitation()` is now a pure function — no DOM, no side effects, easy to unit test
- `validatePayload()` can be reused anywhere you need to check form completeness
- `displayCitation()` and `displayValidationWarning()` only touch the DOM — they know nothing about formatting
- `generate()` becomes a clean orchestrator that just coordinates the steps

## Problem 2: `fetchAndFill()` 
 
### Before — Fetch, Parse, Fill, and Generate All at Once
 
```javascript
async function fetchAndFill() {
    const url = (urlEl.value || '').trim();
    if (!url) { alert('Enter a URL first.'); return; }
 
    fetchBtn.textContent = 'Fetching…';
    fetchBtn.disabled = true;
 
    try {
        // Responsibility 1: Fetch metadata from server
        let meta = await fetchMetadataFromServer(url);
 
        // Responsibility 2: Fallback — fetch raw HTML and parse it
        if (!meta) {
            const html = await tryFetchHtml(url);
            if (!html) {
                alert('Failed to fetch page. CORS or proxy error.');
                return;
            }
            meta = parseMetadataFromHtml(html, url);
        }
 
        // Responsibility 3: Format the date based on current style
        // Responsibility 4: Fill all form fields with extracted data
        if (meta.author) authorEl.value = meta.author;
        if (meta.published) {
            const fmt = formatPublishedDate(meta.published);
            const style = (styleEl && styleEl.value) ? styleEl.value : 'apa';
            dateEl.value = (fmt && fmt[style]) ? fmt[style] : meta.published;
        }
        if (meta.title) titleEl.value = meta.title;
        if (meta.site) sourceEl.value = meta.site;
        urlEl.value = meta.url || url;
 
        // Responsibility 5: Trigger citation generation
        generate();
    } catch (err) {
        console.warn(err);
        alert('Error fetching metadata.');
    } finally {
        fetchBtn.textContent = 'Autofill from URL';
        fetchBtn.disabled = false;
    }
}
```
 
**Problems:**
- The fallback fetching logic is buried inside the same function that fills the form
- Date formatting is scattered inline instead of being handled once
- UI state (button text) is mixed with data-fetching logic
- If the server fetch succeeds but form-filling breaks, you can't tell which step failed
 
---
 
### After — API Chain with Single Responsibilities
 
```javascript
// Responsibility 1: Fetch metadata, try server first then HTML fallback
async function fetchMetadata(url) {
    const serverMeta = await fetchMetadataFromServer(url);
    if (serverMeta) return serverMeta;
 
    const html = await tryFetchHtml(url);
    if (!html) throw new Error('FETCH_FAILED');
 
    return parseMetadataFromHtml(html, url);
}
 
// Responsibility 2: Format the date for the selected citation style
function formatMetaDate(published, style) {
    if (!published) return '';
    const fmt = formatPublishedDate(published);
    return (fmt && fmt[style]) ? fmt[style] : published;
}
 
// Responsibility 3: Fill form fields from metadata object
function fillFormFromMeta(meta, url, style) {
    if (meta.author) authorEl.value = meta.author;
    if (meta.published) dateEl.value = formatMetaDate(meta.published, style);
    if (meta.title)  titleEl.value = meta.title;
    if (meta.site)   sourceEl.value = meta.site;
    urlEl.value = meta.url || url;
}
 
// Responsibility 4: Manage button UI state (separate from data logic)
function setFetchButtonState(loading) {
    fetchBtn.disabled = loading;
    fetchBtn.textContent = loading ? 'Fetching…' : 'Autofill from URL';
}
 
// ORCHESTRATOR: API Chain — fetch → format → fill → generate
function fetchAndFill() {
    const url = (urlEl.value || '').trim();
    if (!url) { alert('Enter a URL first.'); return; }
 
    setFetchButtonState(true);
 
    fetchMetadata(url)                              // Step 1: get metadata
        .then(meta => {
            const style = styleEl.value || 'apa';
            fillFormFromMeta(meta, url, style);     // Step 2: fill form
            generate();                             // Step 3: generate citation
        })
        .catch(err => {                             // Single catch for entire chain
            console.warn('fetchAndFill error:', err);
            const msg = err.message === 'FETCH_FAILED'
                ? 'Failed to fetch page. Try entering fields manually.'
                : 'Error fetching metadata.';
            alert(msg);
        })
        .finally(() => {
            setFetchButtonState(false);             // Always restore button
        });
}
```
 
**What changed:**
- `fetchMetadata()` encapsulates the server-first/HTML-fallback logic — callers don't need to know about it
- `fillFormFromMeta()` only fills the form — it has no idea where the metadata came from
- `setFetchButtonState()` handles only the UI — separating it means it could be reused for any async operation
- The chain (`then → then → catch → finally`) clearly shows the sequential flow, and `.finally()` guarantees the button is always restored even if an error occurs

## Problem 3: `saveToWorksCited()` 
 
### Before — Generate, Validate, Save, and Update UI Together
 
```javascript
function saveToWorksCited() {
    // Responsibility 1: Re-generate the citation
    const citation = generate();
    if (!citation || citation.length < 5) {
        alert('Please generate a citation first.');
        return;
    }
 
    // Responsibility 2: Re-collect payload (already done inside generate())
    const payload = {
        author: safe(authorEl.value),
        date: safe(dateEl.value),
        title: safe(titleEl.value),
        source: safe(sourceEl.value),
        url: safe(urlEl.value)
    };
 
    // Responsibility 3: Build parenthetical citation
    const style = styleEl.value;
    const parenthetical = buildParenthetical({ ...payload, style });
 
    // Responsibility 4: Read from storage, build object, write back
    const saved = JSON.parse(localStorage.getItem(KEY) || '[]');
    const newCitation = {
        citation,
        style: styleEl.value,
        at: Date.now(),
        source: safe(sourceEl.value),
        title: safe(titleEl.value),
        url: safe(urlEl.value),
        author: safe(authorEl.value),
        date: safe(dateEl.value),
        parenthetical: parenthetical
    };
    saved.push(newCitation);
    localStorage.setItem(KEY, JSON.stringify(saved));
 
    // Responsibility 5: Trigger UI updates
    alert('✓ Citation saved to References!');
    loadWorksCited();
    updateNotesSourceSelect();
}
```
 
**Problems:**
- Payload is collected twice — once inside `generate()` and once again here
- Storage logic (read → push → write) is embedded in a UI-facing function
- Testing the save logic would require a real DOM and real localStorage interaction

###  After — Focused Functions with a Clean Orchestrator
 
```javascript
// Responsibility 1: Build a citation record from payload + formatted text
function buildCitationRecord(citation, payload, style) {
    const parenthetical = buildParenthetical({ ...payload, style });
    return {
        citation,
        style,
        at: Date.now(),
        source: payload.source,
        title: payload.title,
        url: payload.url,
        author: payload.author,
        date: payload.date,
        parenthetical
    };
}
 
// Responsibility 2: Append a record to the saved citations list
function appendToStorage(record) {
    const saved = JSON.parse(localStorage.getItem(KEY) || '[]');
    saved.push(record);
    localStorage.setItem(KEY, JSON.stringify(saved));
}
 
// Responsibility 3: Notify the user a citation was saved (UI only)
function notifyCitationSaved() {
    alert('✓ Citation saved to References!');
}
 
// ORCHESTRATOR: Coordinates save workflow
function saveToWorksCited() {
    const citation = generate();
    if (!citation || citation.length < 5) {
        alert('Please generate a citation first.');
        return;
    }
 
    const payload = collectFormPayload();
    const style = styleEl.value;
 
    const record = buildCitationRecord(citation, payload, style);  // Step 1
    appendToStorage(record);                                        // Step 2
    notifyCitationSaved();                                          // Step 3
    loadWorksCited();                                               // Step 4
    updateNotesSourceSelect();                                      // Step 5
}
```
 
**What changed:**
- `buildCitationRecord()` is a pure function — takes inputs, returns an object, no side effects
- `appendToStorage()` only deals with localStorage — completely separate from the UI
- `saveToWorksCited()` reads as a clear step-by-step sequence and delegates all real work to workers
- `collectFormPayload()` is now shared between `generate()` and `saveToWorksCited()` instead of duplicated

## The Full Pattern in Summary
 
Here's how the three principles come together across the refactored citation generator:
 
```
fetchAndFill()              ← Orchestrator
    │
    ├── fetchMetadata()     ← Worker: handles HTTP + fallback logic
    ├── fillFormFromMeta()  ← Worker: fills form fields
    ├── generate()          ← Orchestrator (called as a step)
    │       ├── collectFormPayload()    ← Worker
    │       ├── validatePayload()      ← Worker
    │       ├── formatCitation()       ← Worker (pure function)
    │       ├── displayCitation()      ← Worker (DOM only)
    │       └── displayValidationWarning() ← Worker (DOM only)
    │
    .catch()                ← Single error handler for the chain
    .finally()              ← Guaranteed cleanup (button state)
 
 
saveToWorksCited()          ← Orchestrator
    │
    ├── generate()          ← reused from above
    ├── collectFormPayload() ← reused from above
    ├── buildCitationRecord() ← Worker (pure function)
    ├── appendToStorage()   ← Worker (storage only)
    └── notifyCitationSaved() ← Worker (UI only)
```
 
---
 
## Key Takeaways
 
| Concept | What It Means in This Refactor |
|--------|-------------------------------|
| **SRP** | `formatCitation()` only formats. `validatePayload()` only validates. No function does both. |
| **API Chaining** | `fetchAndFill()` uses `.then().catch().finally()` to make the async steps readable and safe |
| **Orchestrator** | `generate()`, `saveToWorksCited()`, and `fetchAndFill()` coordinate — they don't implement |
| **Worker** | `fetchMetadata()`, `appendToStorage()`, `formatCitation()` implement — they don't coordinate |
| **Pure Functions** | `formatCitation()` and `buildCitationRecord()` take inputs and return outputs with no side effects — they are easiest to test |
